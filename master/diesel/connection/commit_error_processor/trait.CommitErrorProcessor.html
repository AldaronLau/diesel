fn process_commit_error(&self, error: Error) -> CommitErrorOutcome

Returns the status of the transaction following an error upon commit. When any of these kinds of error happen on COMMIT, it is expected that a ROLLBACK would succeed, leaving the transaction in a non-broken state. If there are other such errors, it is fine to add them here.
